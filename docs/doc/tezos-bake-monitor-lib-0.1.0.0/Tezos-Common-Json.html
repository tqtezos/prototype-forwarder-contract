newtype StringEncode a Source #

JavaScript compatibility forces us and the Tezos JSON/RPC to encode any integer with more than 32 bits into a decimal string format. StringEncode helps us safely represent a range of different types for which we must always respect this property.

type TezosPositiveBigNum = StringEncode Natural Source #

WARNING: Due to a major oversight on the part of Tezos Core, this type is incorrectly named. It represents _non-negative_ integers (including zero), not _strictly positive_ integers. Therefore the Haskell Natural type is indeed the correct counterpart.
